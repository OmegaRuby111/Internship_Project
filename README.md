# AI-Driven Stock Advisory System (RAG + LLM)

## Overview

This project implements an **AI-powered stock advisory system** focused on the **FMCG sector**. It combines **Exploratory Data Analysis (EDA)** with a **Retrieval-Augmented Generation (RAG)** pipeline using a **Large Language Model (LLM)** to answer user queries about stocks.

The system enables users to query financial insights and receive **data-grounded, context-aware responses**.

---

## Objective

To identify and recommend the **best-performing stocks** based on user-specified financial indicators from a curated dataset of **103 FMCG companies**.

---

## Key Features

* Data cleaning and normalization of financial metrics
* Composite scoring and classification of stocks (Good / Average / Bad)
* Semantic search using vector embeddings
* LLM-powered chatbot for natural language querying
* RAG pipeline for accurate, context-based responses

---

## Methodology

### 1. Exploratory Data Analysis (EDA)

* Filtered dataset to include only FMCG sector companies
* Selected key financial metrics:

  * ROCE (Return on Capital Employed)
  * EPS (Earnings Per Share)
  * D/E Ratio
  * Free Cash Flow Yield
  * Price/FCF
  * Price/Sales
  * OPM (Operating Profit Margin)
* Normalized all metrics to a **0–100 scale**
* Generated a **composite score** for each stock
* Categorized stocks into:

  * Good
  * Average
  * Bad

---

### 2. Data Transformation

* Converted cleaned dataset into **JSONL format**
* Ensured compatibility with LLM-based pipelines

---

### 3. Embedding & Indexing

* Transformed records into **vector embeddings**
* Used **FAISS** for efficient similarity search

---

### 4. RAG Pipeline

* User query → embedded into vector space
* Top 5 relevant stock records retrieved
* Context + query passed to LLM
* LLM generates final response

---

### 5. LLM Integration

* Model used: **Mistral-7B-Instruct-v0.2**
* Integrated via:

  * Hugging Face Transformers
  * LM Studio API

## How It Works

1. User inputs a query (e.g., *“Which FMCG stock has high ROCE and low debt?”*)
2. Query is embedded into vector space
3. Top relevant stocks retrieved via FAISS
4. Context passed to LLM
5. LLM generates a structured, human-readable answer
